
## Code Style Rules
- Never add inline comments unless explicitly requested
- Prefer clear, descriptive variable and function names
- Explanations must always be outside of code blocks
- Follow existing TypeScript/JavaScript conventions
- Keep functions short and logically structured

## Workflow Rules
- Only generate or modify files in: src/, site/, tests/
- Do not create unnecessary Markdown files (checklists, summaries, explanations) unless explicitly requested
- Do not modify test-results/, pw-report-json/, pw-report-html/, or report/generatedReports/ unless explicitly requested
- Respect existing folder structure and naming conventions
- All code must be ready to run with the existing Playwright framework
